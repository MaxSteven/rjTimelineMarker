add(frame, color, comment)¶

Add a marker based on the provided arguments. If the frames are already marked this information will be overwritten.

Parameters:
  • frame (int) – Frame number
  • color (list) – Color list ( RGB )
  • comment (str) – Tooltip comment
addCallbacks()¶

Add callbacks that will clear all marker information on the timeline every time a new file is created or a file is opened. It also adds callback to determine if the markers are sliding.

addFromUI()¶

Get the frame, color and comments arguments from the UI and add marker(s).

clear()¶

Remove all markers.

deleteLater()¶

Subclass the deleteLater function to first remove the callback, this callback shouldn’t be floating around and should be deleted with the widget.

draw()¶

Take all the marker information and fill in the QWidget covering the timeline. This function will be called by the update and paintEvent function.

event(event)¶

Subclass the event function in order to capture the ToolTip event. The hovered frame is calculated and checked to see if it is marked and commented, if so the toolTip will show.

getDataFromFrame(frame)¶

Get data from frame, this includes the index within the lists, the color and comment.

Parameters:frame (int) – Frame to get data from
Returns:index, frame, color, comment
Return type:tuple
paintEvent(event)¶
pressCommand(*args)¶

Press callback on the timeline, this callback registers the current selected frames, if the user settings determine that the frame range is not important ( no automated shifting of markers ), no range will be stored.

readFromCurrentScene(*args)¶

Marker data can be stored in the Maya’s scenes themselves, the fileInfo command is used for this and the data is stored under the “timelineMarker” argument. This data can be decoded with json and split it the frames, colours and comments.

releaseCommand(*args)¶

Release callback on the timeline, together with the press command the difference can be extracted and the markers can be moved accordingly. Theuser settings will be checked if the moving of markers is appropriate.

remove(frame)¶

Remove a marker based on the provided arguments.

Parameters:frame (int) – Frame number
removeCallbacks()¶

Remove Callbacks.

removeFromUI()¶

Get the frame, color and comments arguments from the UI and remove marker(s).

saveToCurrentScene()¶

Get all the marker information ( frames, comments and colors ) and store this with the fileInfo command in the maya file. Data is stored under the “timelineMarker” argument.
